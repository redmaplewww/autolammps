# LAMMPS 历史经验提炼

本文件不是原始对话归档，而是从迁移资料中的历史对话、每日记忆、纠错记录、review 记录中提炼出的可复用经验。

## 一、已提炼出的高价值主题

### 1. 继续计算前先核验来源文件

- 从 `restart` 或 `data` 文件续算前，不能默认中间文件是正确的
- 必须核对：
  - 文件来源
  - 生成时间
  - box 尺寸是否与建模预期一致
- 特别是多层膜、界面和含真空层结构，必须检查异常大的盒子尺寸

### 2. 应变定义必须冻结初始盒长

- 工程应变计算不能把初始盒长写成随 `lx` 动态变化的 equal-style 变量
- 正确做法是先保存初始盒长，再基于固定初值计算应变

### 3. reviewer gate 不是可选项

- 高风险任务必须审查
- 特别是：
  - ReaxFF / COMB3
  - 新势函数首次使用
  - 多步复杂工作流
  - 用户明确要求审查的任务

### 4. 失败案例必须回写，不然经验无法积累

- 历史资料反复强调：
  - failed / blocked 案例要优先复盘
  - 新失败模式要更新 playbook
  - 错误要转成执行前规则

### 5. 文件名和格式标签不能完全信任

- 历史 review 中出现过“文件名写的是 Al2O3，实际内容却是 MgO”的情况
- 对材料文件、CIF 转换结果、Atomsk 输出，不能只看文件名，必须核查化学式、空间群、盒子和 type 映射

## 二、从每日记忆中抽出的流程性经验

### 1. 参数锁定

- WF-02 确定的势函数配置，WF-03A 不应擅自改动
- 若需要改动，应回退到势函数阶段重新确认

### 2. 大输入/大日志处理

- 超长输入、超长日志需要截断或分块
- 不应一次性把全部大文件送入分析上下文

### 3. 长任务必须保存状态

- 历史工作流明显依赖状态包、版本记录、决策记录和 retry 记录
- 这说明 LAMMPS 任务经常跨天、跨轮次，需要持续追踪

## 三、当前使用原则

- 本文件只存“已经提炼后的经验”，不存原始会话原文
- 若某条经验已经足够稳定，应同步进入：
  - `knowledge/memory/confirmed-lessons.md`
- 若仍需观察，应进入：
  - `knowledge/memory/pending-lessons.md`
