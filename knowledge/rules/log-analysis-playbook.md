# LAMMPS 大日志分析流程

本文件定义超大 `log.lammps` 的最小分析流程，避免一次性整体读取导致低效或超时。

## 一、默认分析顺序

### 1. 初始化区段

优先检查：

- `units`
- `atom_style`
- `boundary`
- `pair_style`
- `pair_coeff`
- `read_data` / `read_restart`

目的：

- 先判断体系设置是否合理

### 2. 错误与警告区段

优先抓取：

- `ERROR`
- `WARNING`
- `Lost atoms`
- `Bond atoms missing`
- `Non-numeric`
- `qeq`

目的：

- 先定位异常根因，再决定是否需要深入热力学段

### 3. 热力学表头与代表性区段

优先抓取：

- thermo header
- 开头若干行
- 中段若干行
- 末段若干行

目的：

- 判断温度、压力、能量、体积是否稳定

### 4. 结尾区段

优先检查：

- run 结束语句
- timing summary
- 最后一次 thermo 输出
- 最终错误或 warning

目的：

- 判断任务是否真正完成，以及完成质量如何

## 二、推荐区段

- 开头：前 80 到 120 行
- 结尾：后 120 到 200 行
- 错误命中：命中行附近各取 5 到 10 行
- thermo：每个 thermo 表头附近取样

## 三、结果输出要求

分析结论中必须说明：

- 使用了哪些区段
- 是否只做了抽样分析
- 是否仍需补读完整日志或某一段详细内容

## 四、使用原则

- 超大日志优先做区段分析，不默认全文硬读
- 若区段分析已足够定位问题，可直接给出诊断
- 若区段之间矛盾，再扩大读取范围
