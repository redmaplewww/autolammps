# Scientific Plotting Skill

This skill turns scientific data into reproducible, publication-ready figures. It is inspired by the
best practices from high-star scientific visualization and chart skills, but is specialized for local
Python research workflows and simulation post-processing.

## Non-negotiable rules

1. **Preserve scripts.** Every figure must be generated by a saved script, never by an unrecorded
   interactive command. For project work, save scripts under `scripts/` and outputs under `outputs/`.
2. **Use data-driven plots.** Read CSV/dat files explicitly, validate columns, and save derived CSVs
   when joins, filters, binning, or aggregations are performed.
3. **Use publication-safe styling.** Use readable fonts, labeled axes with units, appropriate figure
   size, colorblind-safe palettes, and clean layouts without chart junk.
4. **Export reproducibly.** Save at least PNG plus a vector format (`pdf` or `svg`) for line charts.
   Use TIFF/PNG for raster-heavy outputs.
5. **Record provenance.** Generate a manifest for multi-figure tasks with input files, script paths,
   output paths, figure dimensions, row counts, and skipped reasons.

## Default workflow

1. Read the request and identify figure purpose: exploration, report, manuscript, or final publication.
2. Inventory input files and columns before plotting.
3. Choose the right chart type using `references/chart-selection.md`.
4. Select plotting backend:
   - `matplotlib` for publication/static figures and precise multi-panel layouts.
   - `seaborn` for statistical distributions, faceting, confidence intervals, and quick EDA.
   - `plotly` for interactive HTML, hover inspection, dashboards, or 3D/scientific exploration.
5. Apply style from `scripts/publication_style.py` or implement equivalent rcParams.
6. Generate figures and all intermediate CSVs from a saved script.
7. Export required formats and write `plot_manifest.json` for project-scale work.
8. Report output paths, data coverage, and any limitations.

## Figure quality standards

- Axis labels must include units when units are known.
- Legends must be clear, non-overlapping, and unnecessary legends should be removed.
- Use sentence case labels, not all caps.
- Use panel labels `A`, `B`, `C` for multi-panel figures.
- Use consistent fonts, line widths, marker sizes, palettes, and axis scales across related panels.
- Bar charts should start at zero unless there is a scientifically justified reason.
- Avoid 3D effects, shadows, decorative gradients, and `jet`/`rainbow` colormaps.
- For continuous maps, prefer `viridis`, `plasma`, `cividis`, or a physically meaningful diverging map.
- For categorical series, prefer Okabe-Ito or seaborn colorblind palettes.

## Statistical rigor

When plotting experimental or stochastic data:

- Show individual data points when feasible.
- Define error bars as SD, SEM, or CI in labels/captions.
- Include sample size `n` when available.
- Preserve raw data or summary CSV used to make the plot.
- Add significance markers only if the statistical test and p-values are known or computed.

## Export requirements

Default exports:

- `figure.png` at 300 DPI for reports.
- `figure.pdf` or `figure.svg` for vector manuscript figures.
- `figure.tiff` only when required by journals or raster-heavy images.

Recommended sizes:

- Single-column paper figure: `3.5 x 2.5 in` to `3.5 x 3.5 in`.
- Double-column paper figure: `7.0 x 4.0 in` to `7.0 x 6.0 in`.
- Dashboard/composite figure: choose size based on panel count, but keep labels readable at final size.

## Backend-specific rules

### Matplotlib/seaborn

- Prefer `constrained_layout=True` or `fig.tight_layout()` after checking legends/colorbars.
- Use `GridSpec` for complex panel layouts.
- Remove top/right spines for simple statistical charts unless boxed axes are needed.
- For heatmaps, include a labeled colorbar.
- For multi-series lines, combine color with markers or line styles when grayscale compatibility matters.

### Plotly

- Use Plotly Express for quick DataFrame-based charts.
- Use Graph Objects for complex multi-trace, 3D, subplots, and fine-grained layout control.
- Save interactive HTML with `fig.write_html(...)`.
- Save static output with `kaleido` using `fig.write_image(...)` when available.
- Use `plotly_white` or a clean custom template for scientific outputs.

### Web/API chart skills

- External chart APIs are useful for fast visual summaries but should not replace local reproducible
  Python scripts for manuscript figures.
- If external APIs are used, save the request body and returned URL in the manifest.

## Project-scale output contract

For folders with many CSV/dat files and a markdown requirements checklist, create:

```text
postprocess_outputs/
├── scripts/
├── outputs/
├── logs/
├── plot_manifest.json
└── PLOT_REPORT.md
```

`plot_manifest.json` should include:

- source files and row counts
- validated columns
- generated scripts
- derived CSVs
- figure paths and dimensions
- skipped/deferred plots and reasons

## Final checklist

Before reporting completion, verify:

- Script exists for every figure.
- Inputs and derived CSVs are traceable.
- Axes are labeled with units where possible.
- Fonts are readable at final size.
- Colors are colorblind-safe and not `jet`/`rainbow`.
- Multi-panel labels are present and consistent.
- Error bars/statistics are defined when shown.
- PNG plus vector exports exist for publication plots.
- Manifest/report exists for multi-figure tasks.
